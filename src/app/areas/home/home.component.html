  • Problem
    • One component and one template that renders all the person specific layouts
    • A lot of if-statements throughout the template making it hard to read
    • Pros
      • Nothing needs to change
      • One component that has all the shared functions
    • Cons
      • All the if-statements making if hard to read and even harder to understand how the design looks like by just looking at the template
      • One component that has some functions that are not used for all persons
      • Easy to change design/layout for the wrong person
  • Solution 1 - split up into separate components
    • One component + template for each person
    • Extends a base component for shared functions
    • Pros
      • Having a base component that can have shared functions
      • Easy to read template since every template is just for that person
      • Easy to add a new person - just create a new comp
      • Hard to accidentally change a persons layout when you don't mean to
    • Cons
      • A LOT of repeating html in the templates
      • Two new file for every new person
  • Solution 2 - ng-content with parent and child comps
    • One component + template for each person
    • Extends a base component for shared functions and common layout/design
    • Pros
      • Having a base component that can have shared functions
      • Pretty easy to read template since the templates only contains the unique layout to that person
      • Easy to add a new person - just create a new comp
      • Hard to accidentally change a persons layout when you don't mean to
      • More DRY than solution 1
      • ng-content is used in it's intended way here, to create a child comp that only contains template layout specific to that comp, and letting the parent comp have the layout that is shared between persons
    • Cons
      • Will require more work to be rewritten to this way
      • A lot of components - which is both a pro+con
      • Couldn't get showWarning signal to work in the person components
  • Solution 3 - ng-template
    • One component and one template (just like the problem)
    • But we use ng-templates (furthest down in the .html file) to conditionally render specific person layouts
    • Pros
      • Will require less work to be rewritten to this way
      • One component that has the shared functions
      • Pretty to read HTML since we put the ng-templates at the bottom
      • Easy to add a new person - just create a new ng-template
      • Hard to accidentally change a persons layout when you don't mean to
    • Cons
      • Everything in one component - makes the template feel messy
      • ng-template isn't really used in it's intended way here, since it's not repeating a specific layout across the component - every ng-template only contains unique layout for that specific person
